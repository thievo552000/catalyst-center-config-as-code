# Catalyst Center IaC Data

This repository stores exported Cisco Catalyst Center configuration as code. Exports are generated by the Catalyst Center IaC MCP server using read-only playbook config generator tools.

## Layout

```text
data/
  <cluster_name>/
    latest/
      site_playbook_config.yml
      inventory_playbook_config.yml
      ...
    history/
      <run_timestamp>/
        site_playbook_config.yml
        inventory_playbook_config.yml
        ...
reports/
  scale_report_<cluster_name>_<run_timestamp>.csv
```

## Meaning

- `data/<cluster_name>/latest/` is the newest complete or partial snapshot for quick review.
- `data/<cluster_name>/history/<run_timestamp>/` is the immutable snapshot for one export run.
- `reports/` stores one scale-test CSV report per export run.

Each generator flow writes one YAML file. During an export, each completed flow is checkpointed to both `latest/` and `history/<run_timestamp>/`, then appended as one row in the run CSV before the next flow starts.

## Safety

- Store only sanitized configuration outputs and scale-test reports.
- Do not commit MCP server artifacts, raw logs, credentials, API keys, certificates, or local setup notes.
- Keep raw runtime artifacts in the MCP server workspace, not in this repository.
